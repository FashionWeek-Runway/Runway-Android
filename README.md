### Git Conventions

* Feat : 새 기능 (new feature)
* Fix : 버그 수정
* Docs : 문서 수정
* Style : 코드 스타일 관련 수정 (포맷팅 등 실제 코드 변화는 아닌 경우)
* Refactor : 리팩토링 작업
* Test : 테스트 코드의 추가나 테스트 코드에 대한 리팩토링 작업 (메인 코드는 변경 X)
* Chore : 빌드와 관련된 것들을 업데이트하는 경우
* Design : UI 변경 작업
* Comment : 주석의 작성이나 변경
* Rename : 파일, 폴더, 패키지 등의 이름만 수정하거나 옮기기만 한 경우
* Remove : 파일 삭제만 한 경우
